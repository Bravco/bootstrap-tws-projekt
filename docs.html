Inštalácia

Predpoklady

  • Node.js - v18.0.0 alebo novší
  • Textový editor - Odporúčame Visual Studio Code s rozšírením Volar
  • Terminál - Na spustenie príkazov Nuxt

Otvorte terminál (ak používate Visual Studio Code, môžete otvoriť integrovaný terminál) a pomocou nasledujúceho príkazu vytvorte nový štartovací projekt:

npx nuxi@latest init <názov projektu>

Otvorte priečinok projektu v kóde Visual Studio:

code <názov projektu>

Alebo zmeňte adresár na svoj nový projekt z vášho terminálu:

cd <názov projektu>

Nainštalujte závislosti:

npm install

Vývojový server

Teraz budete môcť spustiť svoju aplikáciu Nuxt v režime vývoja:

npm run dev -- -o

Konfigurácia

V predvolenom nastavení je Nuxt nakonfigurovaný tak, aby pokryl väčšinu prípadov použitia. Súbor nuxt.config.ts môže prepísať alebo rozšíriť túto predvolenú konfiguráciu.

Nuxt Konfigurácia

Súbor nuxt.config.ts sa nachádza v koreňovom adresári projektu Nuxt a môže prepísať alebo rozšíriť správanie aplikácie.

Minimálny konfiguračný súbor exportuje funkciu defineNuxtConfig obsahujúcu objekt s vašou konfiguráciou. Pomocník defineNuxtConfig je globálne dostupný bez importu.

export default defineNuxtConfig({
  // Tvoja Nuxt konfigurácia
})

Aplikačná Konfigurácia

Súbor app.config.ts, ktorý sa nachádza v zdrojovom adresári (štandardne koreňový adresár projektu), sa používa na odhalenie verejných premenných, ktoré je možné určiť v čase zostavovania. Na rozdiel od možnosti runtimeConfig tieto nemožno prepísať pomocou premenných prostredia.

Minimálny konfiguračný súbor exportuje funkciu defineAppConfig obsahujúcu objekt s vašou konfiguráciou. Pomocník defineAppConfig je globálne dostupný bez importu.

export default defineAppConfig({
  title: 'Ahoj Nuxt',
  theme: {
    dark: true,
    colors: {
      primary: '#ff0000'
    }
  }
})

runtimeConfig vs. app.config

Vlastnosť runtimeConfig app.config
Strana klienta Hydratovaná Zviazaná
Premenné prostredia ✅ Áno ❌ Nie
Reaktívny ✅ Áno ✅ Áno
Typová podpora ✅ Čistočná ✅ Áno
Neprimitívne JS typy ❌ Nie ✅ Áno

Pohľady

app.vue

V predvolenom nastavení bude Nuxt považovať tento súbor za vstupný bod a vykreslí jeho obsah pre každú cestu aplikácie.

<template>
  <div>
    <h1>Vitaj na stránke!</h1>
  </div>
</template>

Komponenty

Väčšina komponentov sú opakovane použiteľné časti používateľského rozhrania, ako sú tlačidlá a ponuky. V Nuxte môžete tieto komponenty vytvoriť v adresári component/ a budú automaticky dostupné vo vašej aplikácii bez toho, aby ste ich museli explicitne importovať.

app.vue

<template>
  <div>
    <h1>Vitaj na stránke!</h1>
    <AppAlert>
      Toto je automaticky importovaný komponent.
    </AppAlert>
  </div>
</template>

components/AppAlert.vue

<template>
  <span>
    <slot/>
  </span>
</template>

Stránky

Stránky predstavujú zobrazenia pre každý konkrétny vzor trasy. Každý súbor v adresári pages/ predstavuje inú cestu zobrazujúcu jeho obsah.

Ak chcete použiť stránky, vytvorte súbor pages/index.vue a pridajte komponent do app.vue (alebo odstráňte app.vue pre predvolenú položku). Teraz môžete vytvoriť ďalšie stránky a ich zodpovedajúce trasy pridaním nových súborov do adresára pages/.

pages/index.vue

<template>
  <div>
    <h1>Vitaj na stránke!</h1>
    <AppAlert>
      Toto je automaticky importovaný komponent.
    </AppAlert>
  </div>
</template>

pages/about.vue

<template>
  <section>
    <p>Táto stránka bude zobrazená na /about ceste.</p>
  </section>
</template>

Aktíva

Nuxt používa dva adresáre na spracovanie aktív, ako sú šablóny so štýlmi, fonty alebo obrázky.

  • Obsah adresára public/ sa poskytuje v koreňovom adresári servera tak, ako je.
  • Adresár assets/ obsahuje podľa konvencie každé aktívum, ktoré chcete, aby nástroj na zostavovanie (Vite alebo webpack) spracoval.

public/ adresár

Adresár public/ sa používa ako verejný server pre statické aktíva verejne dostupné na definovanej adrese URL vašej aplikácie.

Súbor v adresári public/ môžete získať z kódu vašej aplikácie alebo z prehliadača podľa koreňovej adresy URL /.

Napríklad

Napríklad odkazovanie na súbor s obrázkom v adresári public/img/, ktorý je dostupný na statickej adrese URL /img/nuxt.png:

<template>
  <img src="/img.nuxt.png" alt="Spoznaj Nuxt 3"/>
</template>

assets/ adresár

Nuxt používa Vite (predvolené) alebo webový balík na zostavenie a spojenie vašej aplikácie. Hlavnou funkciou týchto nástrojov na zostavovanie je spracovanie súborov JavaScript, ale môžu byť rozšírené pomocou doplnkov (pre Vite) alebo zavádzačov (pre webpack) na spracovanie iného druhu aktív, ako sú šablóny so štýlmi, fonty alebo SVG. Tento krok transformuje pôvodný súbor hlavne na účely výkonu alebo ukladania do vyrovnávacej pamäte (ako je minifikácia šablón štýlov alebo zrušenie platnosti vyrovnávacej pamäte prehliadača).

Podľa konvencie používa Nuxt na ukladanie týchto súborov adresár assets/, ale pre tento adresár neexistuje žiadna funkcia automatického skenovania a môžete preň použiť akýkoľvek iný názov.

V kóde vašej aplikácie môžete odkazovať na súbor nachádzajúci sa v adresári assets/ pomocou cesty ~/assets/.

Napríklad

Napríklad odkazovanie na súbor s obrázkom, ktorý sa spracuje, ak je zostavovací nástroj nakonfigurovaný na spracovanie tejto prípony súboru:

<template>
  <img src="~/assets/img/nuxt.png" alt="Spoznaj Nuxt 3"/>
</template>
